Heroes Battle Project

Проект реализует ключевые алгоритмы для пошаговой стратегии: генерацию армии, симуляцию боя, поиск целей и поиск пути.

Реализованные алгоритмы

Генерация армии (GeneratePresetImpl)
Алгоритм: Жадный алгоритм.

Сортируем все доступные типы юнитов по критерию эффективности: (Атака / Стоимость) + (Здоровье / Стоимость).

Набираем юнитов начиная с самых эффективных, пока хватает очков и не превышен лимит (11 шт. одного типа).

Координаты выбираются случайным образом в заданной зоне (для компьютера: x=[0..2]), с проверкой на коллизии через HashSet.

Сложность: O(N * log N) — определяется сортировкой списка типов юнитов.

Симуляция боя (SimulateBattleImpl)
Алгоритм: Пошаговая симуляция.

В каждом раунде список живых юнитов сортируется по Атаке (убывание), затем по Здоровью.

Юниты ходят по очереди. Если юнит погибает во время раунда, он пропускает ход.

Бой идет до полного уничтожения одной из армий.

Сложность: O(N^2 * log N) в худшем случае для всего боя (где N — число юнитов), так как сортировка происходит каждый раунд. Для одного раунда сложность O(N * log N).

Поиск целей для атаки (SuitableForAttackUnitsFinderImpl)
Алгоритм: Перебор с проверкой блокировки.

Проходим по всем юнитам целевой армии.

Для каждого юнита проверяем, есть ли другой юнит в той же строке (Y), который находится "перед" ним (ближе к противнику по оси X).

Если препятствий нет — юнит добавляется в список доступных целей.

Сложность: O(N^2) — так как для каждого юнита мы проверяем список всех остальных юнитов на наличие препятствия. При малом N (до ~50) работает мгновенно.

Поиск пути (UnitTargetPathFinderImpl)
Алгоритм: BFS (Поиск в ширину).

Строится граф поля 27x21.

Препятствиями считаются все живые юниты, кроме атакующего и цели.

Используется очередь для обхода клеток "волной" до нахождения цели.

Путь восстанавливается через карту родителей (cameFrom).

Сложность: O(W * H), где W и H — ширина и высота поля. Алгоритм посещает каждую клетку не более одного раза.
